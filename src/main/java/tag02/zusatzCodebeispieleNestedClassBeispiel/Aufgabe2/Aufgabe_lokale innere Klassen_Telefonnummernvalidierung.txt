Hintergrund: 
Java kennt vier Arten von inneren Klassen.
In dieser Aufgabe sollen Sie eine lokale innere Klasse in einer Methode der äußeren Klasse definieren.
Das hier genutzte Prinzip verdeutlicht, wie anonyme innere Klassen später auch funktionieren werden.
Wir erschaffen uns "on the fly" eine innere Klasse und verwenden diese dann ausschließlich im Scope der umgegebenden
Methode oder des umgegebenden Konstruktors.

1a)Implementieren Sie eine Klasse namens TelefonSystem.

Die Klasse TelefonSystem hat folgende Member:

Keine

Methoden:
- eine validiereTelefonnummer Methode, die in ihrer Parameterliste eine Variable telefonnummer vom Typ String entgegennimmt
und void zurückgibt.

Die Methode validiereTelefonnummer(String telefonnummer)der äußeren TelefonSystemklasse definiert nun im Methodenrumfp ein innere
Klasse:
-eine innere Klasse namens Telefonnummer mit folgenden Attributen:

Attribute (alle geschützt):
-formatierteTelefonnummer vom Typ String(diese Variable soll 
fest auf null initialisiert werden.)

CTOR:
-Der Telefonnummer-Konstruktor empfängt in seiner Parameterliste 
ein Argument vom Typ String namens telefonnummer und
initialisiert damit eine lokale Variable namens aktuelleNummer vom Typ String. 
(Diese lokale Variable soll nur innerhalb des Konstruktorrumpfes sichtbar sein.)

-Der Konstruktor sorgt auch für die Validierung der Telefonnummer: 
die aktuelleNummer muss genau 10 Zeichen lang sein (Vergleich mit der Variablen nummernLaenge). 
Ist die aktuelleNummer 10 Zeichen lang dann wird der lokalen Variable 
formatierteTelefonnummer der Wert von aktuelleNummer zugewiesen.
Ist die aktuelleNummer ungleich Länge 10, dann hat die formatierteTelefonnummer weiterhin den Wert null.

Methoden:
- eine getTelefonnummer Methode, welche den Wert von formatierteTelefonnummer zurückgibt.
- eine gibOriginaleTelefonnummerAus ohne Parameter, welche an der Konsole den Wert der 
original übergebenen telefonnummer(nicht die formatierteTelefonnummer) ausgibt.
- lassen Sie an der Konsole ausgeben, ob die originale Telefonnummer gültig war (Erfolgsmeldung). Wenn 
sie nicht gültig war (Länge != 10, d.h. formatierteTelefonnummer ist null) dann lassen Sie eine Fehlermeldung ausgeben.
